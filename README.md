## Overview

AI-Augmented SOAR automatically enriches Elasticsearch security alerts with:

- **Plain-language summaries** generated by Claude AI for immediate analyst comprehension
- **MITRE ATT&CK classification** (tactic + technique) derived from alert context
- **Asset inventory lookups** to surface business context and criticality
- **Threat intelligence** via VirusTotal and AbuseIPDB IOC checks (opt-in)
- **Structured response playbooks** with immediate actions, investigation steps, and EQL hunt queries
- **Multi-turn analyst chat** for interactive alert investigation

Enriched alerts are written back to Elasticsearch and surfaced via a Kibana plugin dashboard.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| AI | Anthropic Claude (configured via `ANTHROPIC_MODEL`) |
| Backend | Python 3.11, FastAPI, Pydantic v2 |
| Data | Elasticsearch 8.13, Redis 7 |
| Frontend | TypeScript, React 18, @elastic/eui, Kibana Plugin SDK |
| Infrastructure | Docker Compose |

## Setup

### 1. Clone and configure

```bash
git clone https://github.com/lenoshz/ai-augmented-soar.git
cd ai-augmented-soar
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY and ELASTIC_PASSWORD
```

### 2. Start the stack

```bash
docker compose up -d
```

### 3. Bootstrap Elasticsearch

```bash
bash elastic/scripts/bootstrap.sh
```

### 4. Verify health

```bash
curl http://localhost:8000/health
```

### Development mode (hot reload)

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```

## Agent Mode Prompts

Suggested prompts for interacting with the analyst chat feature:

- *"What MITRE technique is most relevant to this alert and how should I investigate it?"*
- *"Is the affected host a high-value target? What are the lateral movement risks?"*
- *"Generate an EQL query to hunt for similar activity across the last 24 hours."*
- *"What are the indicators of compromise I should look for in memory forensics?"*

## Troubleshooting

**Elasticsearch not healthy after startup**
- Increase Docker memory allocation to at least 4 GB
- Check logs: `docker compose logs elasticsearch`

**Enrichment service fails to start**
- Verify `ANTHROPIC_API_KEY` is set in `.env`
- Check logs: `docker compose logs enrichment-service`

**No alerts appearing in Kibana plugin**
- Seed test alerts: `python scripts/seed-test-alerts.py --count 5`
- Verify watcher is active: `curl -u elastic:$ELASTIC_PASSWORD http://localhost:9200/_watcher/watch/ai-augmented-soar-enrichment-trigger`

**Tests failing**
```bash
cd enrichment-service
ANTHROPIC_API_KEY=test pytest tests/ -v
```
